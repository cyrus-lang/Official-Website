# Optional (Not implemented yet.)

`Option<T>` - An enumeration that represents either a value **Some(T)** or no value **None**.
This is the standard way to express optional values in Cyrus without resorting to raw nulls or pointers.
By forcing explicit handling of both cases, it helps avoid null-related errors.

```cyrus
import std::types{Option};

func main() {
    #a: Option<int> = Option.None;   # no value
    a = Option.Some(10);             # contains 10
}
```

# Tuples (Not implemented yet.)

Tuple - A fixed-size, ordered collection of values that may be of different types.
Useful for grouping related values without creating a full struct.

# Range (Not implemented yet.)

Range - Represents a sequence of values with a defined start and end.
Ranges are commonly used in loops and iteration, allowing concise traversal of integer intervals.
They do not allocate or store elements, but instead describe the bounds to iterate over.

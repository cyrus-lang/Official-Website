# Basic Syntax

This article is an introduction to the Cyrus Programming Language. It assumes you already have a basic understanding of programming concepts such as variables, statements, and types. Here, you’ll see how familiar ideas are expressed in Cyrus, while also being introduced to new concepts that make the language unique.

If you haven't installed Cyrus yet, start with the [Getting Started](/en/docs/getting-started/install-compiler-binary) guide before continuing.

## Comments

Comments are pieces of text ignored by the compiler.
They're useful for explaining code, leaving notes, or temporarily disabling parts of a program. Cyrus supports both single-line and multi-line comments.

- Single-line Comments

```cyrus
// This is a single-line comment
var msg: uint8* = "Hello, Cyrus!"; // You can also place it after code
```

- Multi-line Comments

```cyrus
/*
  Outer comment start
  /* Nested comment inside */
  Outer comment end

  Multi-line comments can also be nested!
*/
```

## Const string and Character Literals

String literals in Cyrus are enclosed in double quotes **"xxx"**, while character literals are enclosed in single quotes **'x'**.
Both forms represent sequences of Unicode characters, but they differ in intent and usage:

- String literals produce values of type `uint8*` (a pointer to a byte array), which can hold arbitrarily many characters as a sequence of bytes.
- Character literals produce values of type `uint8`, which can hold only a single byte (0-255). In many languages, this means it can represent ASCII characters or a single byte of UTF-8, but not arbitrary Unicode scalar values that require multiple bytes (like '€' or '你').

```cyrus
var greeting: uint8* = "Hello, Cyrus!";
var letter: uint8 = 'A';
```

### Escapes

Special characters that are difficult to type directly or would otherwise break the syntax can be written using escape sequences. All escape sequences begin with a backslash (\).

Common escapes include:

- **\n** - newline (line feed, U+000A)
- **\t** - horizontal tab (U+0009)
- **\r** - carriage return (U+000D)
- **\b** - backspace (U+0008)
- **\a** - bell/alert (U+0007)
- **\v** - vertical tab (U+000B)
- **\f** - form feed (U+000C)
- **\\** - literal backslash
- **\"** - double quote (inside strings)
- **\'** - single quote (inside chars or strings)

Example:

```cyrus
var poem: uint8* = "Line one\nLine two\nLine three";
var quote: uint8* = "She said: \"Cyrus is great!\"";
var backslash: uint8 = '\\';

var paragraph: uint8* = "Lorem Ipsum is simply dummy text of the printing and typesetting industry.
Lorem Ipsum has been the industry's standard dummy text ever since the 1500s,
when an unknown printer took a galley of type and scrambled it to make a type specimen book.";
```

### Unicode and Emojis

Cyrus supports Unicode characters directly in string and character literals. You can type emojis, accented letters, or any other Unicode characters inside quotes:

```cyrus
var letter: uint8 = 'A';                     // valid
var smile: uint8* = "😇";                    // valid
var greek_alpha: uint8* = "\u03B1";          // valid "α"
var emoji_char: uint8 = '😇';                // NOT VALID
var multibyte_char: uint8 = '\u263A';        // NOT VALID
var rocket: uint8* = "Let's go! \u{1F680}";  // valid "Let's go! 🚀"
```

## Integer Constants

Numeric constants in Cyrus can be written in **decimal**, **hexadecimal**, **octal**, or **binary** formats:

- Decimal (Base 10):

```cyrus
var a = 1234;
```

- Hexadecimal (Base 16, prefix with 0x or 0X):

```cyrus
var b = 0x42EDAA02;
```

- Binary (Base 2, prefix with 0b or 0B):

```cyrus
var c = 0b101010;
```

Numeric literals may include underscores **\_** between digits for readability; these are ignored by the compiler:

```cyrus
var million = 1_000_000;
var hex_val = 0xDE_AD_BE_EF;
```

## Literal Suffixes

Cyrus allows explicit type suffixes on numeric literals to specify their type.
This is useful when the compiler cannot infer the type or when you want to guarantee a specific type.

### Integer Suffixes

Valid integer suffixes include:

- uintptr
- intptr
- usize
- isize
- int8
- int16
- int32
- int64
- int128
- uint8
- uint16
- uint32
- uint64
- uint128

Valid float suffixes include:

- float16
- float32
- float64
- float128

Example:

```cyrus
var a = 1234int64;
var b = 3.14float64;
```

### Default Type of Literals

If you don't specify a type suffix, Cyrus assigns `int32` as the default type for integer literals, and `float64` as the default type for floating-point literals.

Type suffixes can be used to explicitly control the size and signedness of an integer literal, or to select a different floating-point precision.

## Arrays

Arrays are fixed-size, stack-allocated sequences. Size must be known at compile time.

### Explicit and Inferred Types

You can define types explicitly or let the compiler infer them:

```cyrus
// Explicit
var a = int32[3]{1, 2, 3};

// Inferred
const untyped = {1, 2, 3};
```

### Multi-dimensional Arrays

```cyrus
const matrix = int32[2][3] {
    int32[3] {1, 2, 3},
    int32[3] {1, 2, 3},
};
```

## Pointers

Cyrus pointers function similarly to C.

```cyrus
var a: int32 = 10;
var b: int32* = &a;
```

### GEP and Decay

Cyrus simplifies pointer access:

1. **GEP**: You can use indexing on pointers directly (GetElementPtr semantics).

   ```cyrus
   var ptr: int32* = malloc(8);
   ptr[0] = 10;
   ptr[1] = 20;
   ```

2. **Decay**: Arrays automatically decay to pointers when assigned to a pointer type.

   ```cyrus
   const nums = int32[3]{1, 2, 3};
   const ptr: int32* = nums; // Equivalent to &nums[0]
   ```

3. **Pointer to Array**:

   ```cyrus
   const nums = int32[3]{1, 2, 3};
   var ptr: int32[3]* = &nums;

   (*ptr)[0] = 10;
   ```

### Pointer Constness and Mutability

Cyrus distinguishes between the mutability of the pointer itself and the mutability of the data it points to.
If you take a reference to a const value, the resulting pointer is automatically a pointer to a constant (e.g., const int32\*), preventing mutation of the underlying value through that pointer.

```cyrus
pub fn main() {
   const x = 10;

   // immut_ref is a mutable pointer to a constant integer (const int32*)
   var immut_ref = &x;

   // ERROR: cannot modify a value through a 'const int32*'
   *immut_ref = 20;

   printf("%d\n", *immut_ref);

   // However, the pointer itself can be reassigned:
   const y = 30;
   immut_ref = &y;
}
```
